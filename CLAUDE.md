設計の正本は `~/ai-context/mv/` の3文書。**本リポジトリには移設・コミットしない**（リポジトリはPublic、正本は作業環境等の内部情報を含むため）。読み取り参照のみ。

## 2. SQLite接続

投入スクリプト・ビルドの双方で、接続ごとに常に `PRAGMA foreign_keys=ON` を固定する（SQLiteの外部キーは接続単位で有効化が必要なため）。

## 3. スコープと承認制

- スコープは**フェーズ2a**（PHASE2.md「2a作業手順」手順1〜7）。2bには入らない。
- DATABASEの未規定事項（関係テーブル・生記述層の主キー構成など）の確定は、決定案と根拠を提示して**依頼者の承認**を得てから実装する。正本ドキュメントへの反映は依頼者が行う。
- GitHub Pagesの有効化・カスタムドメイン設定・最初の公開など、公開に関わる操作は**依頼者の明示の承認**なしに行わない（公開したURL・IDは恒久契約になるため）。
